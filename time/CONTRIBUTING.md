**How Can I Contribute?**

***Reporting Bugs***

Before creating bug reports, please check the existing issues as you might find that you don't need to create one.

-  Use a clear and descriptive title for the issue to identify the problem.
-  Describe the exact steps which reproduce the problem with as many details as possible.
-  Provide specific examples to reproduce the bug.
-  Describe the behavior you observed after following the steps.
-  Explain which behavior you expected to see instead.
-  Suggesting Enhancements
-  Open an issue with a description of the enhancement.
-  Provide a clear and detailed explanation of the suggested enhancement.
-  Explain why the enhancement would be useful to most Getting started with Ruby users.

**Pull Requests**

-  Fork the repo on GitHub.
-  Clone the project to your own machine.
-  Create a new branch (git checkout -b feature/your-feature or fix/your-fix).
-  Make your changes and add tests to cover them.
-  Commit your changes using a descriptive commit message.
-  Push your branch to GitHub.
-  Submit a Pull Request to the main branch.
Note: By submitting a pull request, you agree to allow the project owner to license your work under the same license as that used by the project.

**Style Guides**

Code Style: Please follow the coding style used in the existing codebase (e.g., [ESLint, PEP8, etc.]).
Commit Messages: Keep commit messages concise and descriptive.
